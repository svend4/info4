# СТРАТЕГИЯ РАБОТЫ С КОНТЕКСТОМ И ПРОДОЛЖЕНИЯ ПРОЕКТА

## ЧАСТЬ 1: ЧТО ТАКОЕ КОНТЕКСТ И ТОКЕНЫ

### Что такое токены?

**Токен** — это базовая единица текста, которую обрабатывает Claude. Примерно:
- 1 токен ≈ 0.75 слова (английский)
- 1 токен ≈ 0.5-0.6 слова (русский, из-за кириллицы)
- 1 токен ≈ 4 символа (в среднем)

**Примеры:**
```
"Hello world" = 2 токена
"Привет мир" = 4 токена (русский требует больше)
"function calculateTotal()" = 5 токенов
```

### Что такое "контекст"?

**Контекст (Context Window)** — это "оперативная память" Claude в рамках одного разговора. Всё, что Claude видит и помнит в текущем чате.

**Что входит в контекст:**
1. **Системные инструкции** — правила работы Claude (~50,000 токенов)
2. **История разговора** — все ваши сообщения + ответы Claude
3. **Загруженные файлы** — если вы прикрепили документы
4. **Skills и инструменты** — описания доступных функций

**Лимит контекста:** 200,000 токенов (общий)

### Текущая ситуация

```
═══════════════════════════════════════════════════════════
ИСПОЛЬЗОВАНИЕ КОНТЕКСТА (на момент вашего вопроса)
═══════════════════════════════════════════════════════════

Всего доступно:        200,000 токенов
Использовано:          ~131,000 токенов  
Осталось:              ~69,000 токенов   ← ВОТ ЭТО ВАЖНО!

Распределение использованных токенов:
├─ Системные инструкции: ~50,000 токенов (Claude's rules)
├─ История разговора:    ~60,000 токенов (наша переписка)
├─ Skills metadata:      ~15,000 токенов (описания skills)
└─ Созданные файлы:      ~6,000 токенов  (references)

═══════════════════════════════════════════════════════════
```

### Почему это важно?

**Проблема:** Мы создаём ОЧЕНЬ детальный каталог:
- 87 skills total
- Каждый skill детально = ~7,000 слов = ~10,000 токенов
- 87 × 10,000 = 870,000 токенов ТОЛЬКО на skills!

**Реальность:**
- У нас есть только 69,000 токенов свободных
- Можем описать детально ещё примерно 6-7 skills
- Потом контекст заполнится

### Что происходит когда контекст заканчивается?

❌ **НЕ ПРОИСХОДИТ:**
- Claude НЕ "забывает" предыдущие сообщения
- История НЕ удаляется

✅ **ПРОИСХОДИТ:**
- Claude НЕ МОЖЕТ отправить новое сообщение
- Показывается ошибка "Context limit exceeded"
- Нужно начать НОВЫЙ разговор

## Правила качества

### Обязательно ВКЛЮЧАТЬ:
✓ Примеры кода (минимум 5 блоков)
✓ Конкретные команды пользователя ("Сделай X")
✓ Input/Output примеры
✓ Сравнение с существующими инструментами
✓ Real-world use cases (не абстрактные)

### НЕ ДЕЛАТЬ:
✗ Абстрактные описания без примеров
✗ "Skill может сделать то-то" — ПОКАЗЫВАЙ КАК
✗ Короткие описания (<3000 слов на skill)
✗ Пропуск секций

### Целевой размер:
- Минимум: 5,000 слов / skill
- Оптимально: 7,000-10,000 слов / skill
- Примеры кода: 30-40% объёма

---

## Тон и стиль

### Язык: Русский
- Термины можно на английском (skill, workflow, use case)
- Команды пользователя — естественный русский
- Код — на языке программирования (с комментариями)

### Стиль:
- Практичный, конкретный
- Больше ПРИМЕРОВ, меньше теории
- Как в техническом руководстве, не в академической статье

### Форматирование:
- Используй emoji для приоритетов: 🔴🟡🟢
- Checkbox: ✓ ✗ ❌ ✅
- Код блоки с указанием языка: ```python
- Таблицы где уместно
```

## ЧАСТЬ 5: ГАРАНТИИ КАЧЕСТВА ПРИ ПЕРЕХОДАХ

### Как обеспечить continuity?

**1. Методология — единый стандарт**
- METHODOLOGY.md содержит точную структуру
- Claude в каждом чате читает её перед началом
- Все skills описаны одинаково

**2. Checkpoint файлы — передача контекста**
- Каждый checkpoint содержит summary предыдущего чата
- Claude не теряет context благодаря этим файлам

**3. Progress Tracker — никакие skills не пропущены**
- Видно что описано, что нет
- Checklist формат

**4. Примеры из предыдущих skills**
- В CHECKPOINT файлах есть примеры хороших описаний
- Claude может reference их как образец

**5. Вы — единая точка continuity**
- Вы видите все чаты
- Вы можете указать Claude если он отклонился от стандарта
